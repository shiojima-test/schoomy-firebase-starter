3セットアップ手順

以下の7ステップを順番に進めてください。

1
Firebaseプロジェクトを作る
  1. https://console.firebase.google.com にアクセスする
  2. Googleアカウントでログインする
  3. 「プロジェクトを追加」をクリック
  4. プロジェクト名を入力する（例：schoomy-sensor）
  5. Googleアナリティクスは「オフ」にする（不要です）
  6. 「プロジェクトを作成」をクリック
💡 ヒント：プロジェクト名は何でもOKです。自分がわかる名前をつけてください。
2
Realtime Database を有効化する
  1. 左メニューの「構築」→「Realtime Database」をクリック
  2. 「データベースを作成」をクリック
  3. ロケーション：asia-southeast1（シンガポール）を選ぶ
  4. セキュリティルール：「テストモードで開始する」を選ぶ
  5. 「有効にする」をクリック
⚠️ 注意：テストモードは30日間の期限があります。期限が切れるとデータの読み書きができなくなります。延長方法は「よくあるトラブル」で説明します。
3
firebaseConfig を取得する
  1. プロジェクトのトップ画面で、歯車アイコン → 「プロジェクトの設定」をクリック
  2. ページ下部の「マイアプリ」セクションで、「</>」（Webアプリ）アイコンをクリック
  3. アプリのニックネームを入力する（何でもOK。例：sender）
  4. 「アプリを登録」をクリック
  5. 表示される firebaseConfig のコード全体をコピーする

コピーするのは以下のような部分です：

// これをまるごとコピーします const firebaseConfig = { apiKey: "AIzaSyB1234567890abcdefg", authDomain: "schoomy-sensor.firebaseapp.com", databaseURL: "https://schoomy-sensor-default-rtdb.asia-southeast1.firebasedatabase.app", projectId: "schoomy-sensor", storageBucket: "schoomy-sensor.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef1234567890" };
💡 ヒント：databaseURL は Realtime Database を作ったときに自動で設定されます。もし表示されていない場合は、Realtime Database のページ上部に書いてあるURLをコピーしてください。
4
sender.html を書き換える
  1. ダウンロードした sender.html をメモ帳・VS Code・テキストエディタで開く
  2. 「★ ここを書き換えてください（1/2）」のコメントを探す
  3. firebaseConfig の中身を、STEP 3 でコピーした値に書き換える
  4. 「★ ここを書き換えてください（2/2）」のコメントを探す
  5. DATA_PATH を自分で決めて入力する（例："light" "temperature" "sensor/classroom1"）
  6. ファイルを保存する

書き換え後のイメージ：

// ★ ここを書き換えてください（1/2） const firebaseConfig = { apiKey: "AIzaSyB1234...", ← 自分の値に authDomain: "schoomy-sensor...", ← 自分の値に databaseURL: "https://...", ← 自分の値に // ...省略... }; // ★ ここを書き換えてください（2/2） const DATA_PATH = "light"; ← 自分で決めた名前
💡 ヒント：DATA_PATH は「どこにデータを保存するか」のパス名です。好きな名前でOKですが、sender.html と receiver.html で同じ値にしてください。
5
receiver.html を書き換える
  1. receiver.html をテキストエディタで開く
  2. sender.html とまったく同じ firebaseConfig を貼り付ける
  3. sender.html とまったく同じ DATA_PATH を入力する
  4. ファイルを保存する
⚠️ 重要：sender と receiver で firebaseConfig と DATA_PATH が1文字でも違うと動きません。コピー＆ペーストで揃えるのが確実です。
6
sender.html をChromeで開いて接続する
  1. Google Chromeで sender.html を開く（ファイルをChromeのウィンドウにドラッグ＆ドロップ）
  2. オレンジボードをUSBケーブルでPCに接続する
  3. 画面の「🔌 接続」ボタンをクリック
  4. ポート選択のダイアログが出る → 「CH340」または「USB Serial」と書かれたものを選ぶ
  5. 「接続」をクリック

✅ 「接続中 — データを受信しています」と表示されれば成功！

画面のログに「送信: 1280 raw → light」のように値が流れていきます。

💡 ヒント：ポート選択ダイアログに何も出ない場合は、USBケーブルを一度抜いて差し直し、もう一度ボタンを押してください。
7
receiver.html でリアルタイム受信を確認する
  1. 別のタブまたは別のPCの Chrome で receiver.html を開く
  2. 「現在のセンサー値」が自動で更新されれば成功！
  3. （任意）Firebase Console の Realtime Database を開くと、値がリアルタイムで届いているのも確認できます

✅ sender.html で送った値が receiver.html に表示されればセットアップ完了です！

4よくあるトラブル

Q. ポート選択ダイアログに何も表示されない
USBケーブルを一度抜いて、もう一度差し込んでからリトライしてください。また、SchooMy IDE（Arduino IDE）でプログラムのアップロードが完了しているか確認してください。アップロード中はシリアルポートが占有されます。
Q. sender.html で値は出るが、receiver.html に届かない
firebaseConfig と DATA_PATH が sender と receiver で完全に一致しているか確認してください。1文字でも違うと別のデータベース・別のパスを見てしまいます。
Q. 「PERMISSION_DENIED」エラーが出る
Firebaseのテストモード（セキュリティルール）の30日間の期限が切れています。Firebase Console → Realtime Database →「ルール」タブを開き、以下のように書き換えて「公開」をクリックしてください：
{ "rules": { ".read": true, ".write": true } }
※ これは誰でも読み書きできる設定です。テスト用としてお使いください。
Q. Chrome以外のブラウザで動かない
Web Serial API はGoogle Chrome のデスクトップ版のみで動作します。Safari・Firefox・Edge・スマートフォンのブラウザでは使えません。receiver.html はどのブラウザでも動きますが、sender.html は Chrome 限定です。

⚠️ 注意事項

  • Chrome デスクトップ版限定：sender.html の Web Serial API は Google Chrome（デスクトップ版）でのみ動作します
  • テストモードは30日で期限切れ：Realtime Database のテストモードは30日間有効です。期限後は「よくあるトラブル」の手順でルールを更新してください
  • 長期運用時はセキュリティルールを設定：公開サービスとして使う場合は、Firebase Authentication と組み合わせた適切なセキュリティルールを設定してください
  • HTTPS または localhost：Web Serial API は HTTPS 環境が必要です。ローカルファイル（file://）でも動作しますが、公開時は GitHub Pages 等の HTTPS 環境を使ってください
